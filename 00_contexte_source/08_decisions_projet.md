# Décisions projet

Statut : décisions de cadrage consolidées localement.

## Décisions validées

- Le rapport concerne l’activité entreprise CI1 à l’IMT Nord Europe.
- L’entreprise présentée est Leroux.
- Le service concerné est le service informatique.
- Le rôle de l’étudiant est alternant administrateur systèmes et réseaux.
- L’usage de l’IA sera déclaré dans une annexe dédiée.
- Les fichiers de contexte sont la source de vérité du projet.
- Les consignes IMT et la grille d’évaluation deviennent des contraintes de référence pour tout le rapport.
- L’annexe IA est obligatoire et devra être rédigée avec transparence.
- Le rapport vise 25 à 40 pages hors annexes.
- Les missions principales retenues sont :
  - Microsoft Intune / Android Enterprise ;
  - registre visiteurs Power Apps / SharePoint / Power Automate ;
  - sécurité opérationnelle AD / Microsoft 365.
- Docker `prd-dkr01` reste une mission secondaire ou optionnelle.
- Xibo, uniFLOW et GLPI restent exclus comme missions principales.
- GLPI peut être cité uniquement comme lien, outil consultable ou contexte ponctuel.
- Xibo et uniFLOW peuvent être cités uniquement comme contexte ou activité ponctuelle si nécessaire.

## Décision — Audit du style d’écriture

Les anciens rapports Word de l’étudiant sont utilisés uniquement comme références stylistiques.
Ils ne sont pas utilisés comme sources factuelles pour le rapport CI1 Leroux.
La rédaction doit conserver une logique proche du style observé, tout en corrigeant les fautes, répétitions et maladresses.

## Décision — Utilisation contrôlée des sous-agents

Les sous-agents peuvent être utilisés pour améliorer la qualité des relectures, mais uniquement lorsque la tâche le justifie.

Ils ne rédigent pas indépendamment les grandes parties du rapport.

Ils servent principalement à contrôler :

* conformité IMT ;
* style personnel ;
* cohérence technique ;
* confidentialité ;
* clarté.

L’agent principal reste responsable des modifications finales.

## Décision — Traitement des informations manquantes

Quand une information manque pour rédiger correctement, l’IA ne doit pas inventer.

Elle doit :

1. écrire `[À compléter]` ou `[À vérifier]` dans les fichiers de contexte ;
2. formuler une question claire à poser à l’étudiant ;
3. expliquer pourquoi cette information est nécessaire ;
4. indiquer dans quelle partie du rapport cette réponse sera utilisée.

Cette règle s’applique à toutes les parties du rapport, aux fichiers de contexte, aux contrôles qualité et aux futures relectures.
